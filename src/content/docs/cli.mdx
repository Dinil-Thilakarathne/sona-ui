# CLI Reference

The Sona UI CLI is a tool to help you add components and configure your project.

## Commands

### `init`

Initialize Sona UI in your project. This command creates a `components.json` file, installs dependencies, and configures Tailwind CSS.

<CodeBlockWrapper code="npx @sonacode/sonaui-cli init" language="bash" />

### `add`

Add a component to your project. This command downloads the component source code and places it in your configured components directory.

<CodeBlockWrapper
  code="npx @sonacode/sonaui-cli add [component]"
  language="bash"
/>

**Examples:**

<CodeBlockWrapper
  code={`# Add a single component
npx @sonacode/sonaui-cli add Accordion

# Add multiple components

npx @sonacode/sonaui-cli add Accordion RippleButton`}
language="bash"
/>

### Options

| Option                | Description                                               |
| :-------------------- | :-------------------------------------------------------- |
| `-y`, `--yes`         | Skip confirmation prompts.                                |
| `-o`, `--overwrite`   | Overwrite existing files.                                 |
| `-c`, `--cwd <cwd>`   | The working directory. Defaults to the current directory. |
| `-p`, `--path <path>` | The path to add the component to.                         |

## FAQ

**Where are components installed?**

By default, components are installed in the `src/components` directory (or `components` if `src` is not detected), but this is configurable via `components.json`.

**Do I need the CLI?**

No, the CLI is optional. You can manually copy and paste code from the documentation. However, the CLI speeds up the process by handling dependencies and file placement for you.
